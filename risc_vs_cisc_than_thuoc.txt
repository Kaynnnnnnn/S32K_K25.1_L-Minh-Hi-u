So sánh RISC (Reduced Instruction Set Computer) và CISC (Complex Instruction Set Computer)

I. Điểm giống nhau
- Cả hai đều là kiến trúc tập lệnh (Instruction Set Architecture - ISA) của CPU.
- Đều cung cấp các lệnh cơ bản như: nạp dữ liệu (load), lưu dữ liệu (store), tính toán (add, sub), nhảy (jump)...
- Mục tiêu cuối cùng: làm cho CPU thực hiện chương trình nhanh và hiệu quả hơn.

II. Điểm khác nhau

1. Tập lệnh
- RISC: Ít lệnh, đơn giản, dễ hiểu. Mỗi lệnh thường chỉ làm một việc.
- CISC: Nhiều lệnh, phức tạp. Một lệnh có thể làm nhiều việc (load + tính toán + lưu).

2. Độ dài lệnh
- RISC: Độ dài cố định (ví dụ: 32 bit), nên dễ dự đoán, dễ pipeline.
- CISC: Độ dài không cố định (có lệnh ngắn, lệnh dài), khó pipeline.

3. Chu kỳ xử lý
- RISC: Mỗi lệnh thường chỉ cần 1 chu kỳ CPU → nhanh.
- CISC: Một lệnh có thể mất nhiều chu kỳ để hoàn thành.

4. Truy cập bộ nhớ
- RISC: Chỉ cho phép truy cập bộ nhớ qua lệnh Load/Store.
- CISC: Nhiều lệnh có thể trực tiếp đọc/ghi bộ nhớ.

5. Thành phần hoạt động mạnh hơn
- RISC: Nhấn mạnh vào phần mềm (compiler phải sinh nhiều lệnh nhỏ).
- CISC: Nhấn mạnh vào phần cứng (CPU phải phức tạp để giải mã lệnh).

6. Thiết kế CPU
- RISC: CPU đơn giản, dễ tối ưu tốc độ, tiết kiệm điện → phù hợp thiết bị di động, IoT.
- CISC: CPU phức tạp, tiêu thụ điện nhiều hơn → hay dùng trong PC, laptop, server.

7. Ví dụ
- RISC: ARM, MIPS, RISC-V.
- CISC: Intel x86, AMD.

III. Tóm gọn dễ nhớ
- RISC: Nhiều code hơn, CPU đơn giản, chạy nhanh, ít tốn năng lượng.
- CISC: Ít code hơn, CPU phức tạp, dễ viết nhưng tốn năng lượng và chạy chậm hơn chút.
